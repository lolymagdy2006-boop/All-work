Operation of Relational Database Management Systems

console, client software, web interface - Part 4

Training session: Let's dive into a "Broken Database" troubleshooting scenario. We'll simulate a realistic operational mess: a slow query, a missing index, and a structural bottleneck.

Phase 1: Setting the Stage (The "Messy" Data)

Copy and run this script in your Console or Client Software (DBeaver/Workbench). It creates two tables: customers and orders, but it "forgets" to add any performance optimization.

1. Create Tables

CREATE TABLE customers (
    customer_id INT,
    name VARCHAR(255),
    email VARCHAR(255)
);

CREATE TABLE orders (
    order_id INT,
    c_id INT,          -- This is supposed to be the customer_id
    amount DECIMAL(10,2),
    order_date DATE
);

2. Insert "Messy" Data

INSERT INTO customers VALUES 
(1, 'Alice Smith', 'alice@example.com'),
(2, 'Bob Jones', 'bob@example.com');

INSERT INTO orders VALUES 
(101, 1, 500.00, '2025-01-01'),
(102, 1, 150.00, '2025-01-02'),
(103, 2, 900.00, '2025-01-03');

Phase 2: Operational Troubleshooting

Task 1: Find the Bottleneck (The Client Software)

In DBeaver or MySQL Workbench, run an EXPLAIN on this join:

EXPLAIN SELECT name, SUM(amount)
FROM customers
JOIN orders ON customers.customer_id = orders.c_id
GROUP BY name;

The Problem: You will see "Full Table Scan" (usually in red). Because there are no Primary Keys or Indexes, the database has to read every single row to find a match.

The Fix: Use the GUI to add a Primary Key to customer_id.

Task 2: Structural Integrity (The Console)

The orders table has a column called c_id, but it isn't "linked" to the customers. If you delete a customer, the orders stay behind like ghosts (Orphans).

-- Step 1: Add the Primary Key we missed
ALTER TABLE customers ADD PRIMARY KEY (customer_id);

-- Step 2: Add the Foreign Key to link them
ALTER TABLE orders ADD CONSTRAINT fk_customer 
FOREIGN KEY (c_id) REFERENCES customers(customer_id);

Task 3: Security & Access (The Web Interface)

Imagine a new intern needs to view the total sales but must not see customer emails (GDPR/Privacy).

CREATE VIEW v_sales_summary AS
SELECT c_id, amount FROM orders;

Go to Views and create a "Safe View". Use the Privileges tab to grant the intern access only to the v_sales_summary view, not the raw tables.
